# 동적 라우트

주어진 패턴을 가진 라우트를 동일한 컴포넌트에 매핑해야하는 경우가 자주 있습니다. 예를 들어 동일한 레이아웃을 가지지만 교육과정 이름(courseName)에 따라 각각 다른 데이터를 바인딩하여 렌더링 하는 Course 컴포넌트가 있다고 합시다. 이런 경우 동적으로 라우트를 변경하도록 동적 세그먼트(Dynamic Segment)를 사용하여 다음과 같이 설정할 수 있습니다.







 





// router.js

export default new Router({
  routes: [
    {
      name: 'course',
      path: '/course/:courseName', // :동적 세그먼트
      component: Course
    }
  ]
})

동적 라우팅을 설정하면 아래 나열된 라우트는 모두 동일한 컴포넌트로 연결되어 화면에 표시됩니다.

라우트 컴포넌트
/course/vue-learning-guide Course.vue
/course/nuxt-마스터-가이드 Course.vue
/course/파이어베이스-러닝-가이드 Course.vue

# $route 속성

동적 세그먼트로 설정된 데이터는 Vue 인스터늣의 $route 속성에 매핑됩니다. Vue 컴포넌트는 $route 객체의 params 속성을 통해 전달된 동적 세그먼트 데이터 값에 접근할 수 있습니다.

views/Course.vue



 










<template>
  <div class="course">
    <h1>{{ $route.params.courseName }}</h1>
  </div>
</template>

<script>
export default {
  name: 'Course',
  // ...
};
</script>

# props 속성

라우트 객체에 props 속성 값을 true로 설정하면 페이지 컴포넌트 속성으로 전달 받도록 설정할 수 있습니다.









 




// router.js

export default new Router({
  routes: [
    {
      name: 'course',
      path: '/course/:courseName',
      component: Course,
      props: true
    }
  ]
});

페이지 컴포넌트 Course는 props 속성을 사용해 전달 받을 라우트 매개변수를 설정한 후, 컴포넌트 데이터로 사용할 수 있어 라우트 매개변수를 바인딩 하기 쉬워집니다. ($route.params.courseName ⇒ courseName)

views/Course.vue



 






 



<template>
  <div class="course">
    <h1>{{ courseName }}</h1>
  </div>
</template>

<script>
export default {
  name: 'Course',
  props: ['courseName']
};
</script>
