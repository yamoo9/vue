# 슬롯(Slots)

Vue.js는 컴포넌트 템플릿 재사용성을 향상하기 위해 W3C 웹 컴포넌트 사양 초안에서 영감을 얻어 콘텐츠 배포를 위한 API를 구현했습니다. 표준에 제안된 <slot> 요소를 콘텐츠 배포를 위한 콘센트(concentric plug)로 사용합니다.

NOTE

램 슬롯을 떠올리면 쉽게 이해될 겁니다. 아래 그림을 참고하세요.


# 슬롯을 사용하는 이유

컴포넌트 템플릿 재사용 시 속성을 전달하는 방법 보다, 콘텐츠를 끼워 넣는 방법이 보다 유용합니다.

콘텐츠를 속성으로 전달

<navigation-link url="/profile" content="프로필"/>
1

슬롯을 사용해 콘텐츠 끼워넣기


 


<navigation-link url="/profile">
  프로필
</navigation-link>

# 슬롯 기본 값

컴포넌트 내부 슬롯 요소 안에 콘텐츠를 삽입하면 슬롯의 기본 값이 됩니다. (기본 값 ≒ 대체 콘텐츠)



 



<template>
  <a :href="url">
    <slot>링크 텍스트</slot>
  </a>
</template>

# 이름이 있는 슬롯

이름이 있는 슬롯이란? 슬롯을 좀 더 확장해 사용할 수 있도록 슬롯에 이름을 부여해 사용하는 방법입니다. 즉, 컴포넌트 외부에서 끼워 넣을 콘텐츠를 슬롯 이름으로 구분하여 끼워넣도록 하는 것입니다.

NOTE

햄버거를 만들 때, 이름으로 구분된 위치에 각 재료를 끼워넣어야 한다고 생각하면 이해하기 쉽습니다.



# slot 속성

이해를 돕기 위해 app-card 컴포넌트를 사용할 때 외부에서 콘텐츠를 끼워넣는 예제를 살펴봅시다. 컴포넌트 템플릿에 사용된 슬롯 요소는 총 3개입니다. 그 중 2개 슬롯은 name 속성을 통해 외부에서 끼워넣어질 콘텐츠를 식별하기 위한 이름이 설정되어 있습니다.




 


 


 




<template>
  <div class="app-card">
    <div class="app-card-header">
      <slot name="header"/>
    </div>
    <div class="app-card-content">
      <slot/>
    </div>
    <div class="app-card-footer">
      <slot name="footer"/>
    </div>
  </div>
</template>

# v-slot 디렉티브

slot 속성은 현재 사용을 권장하지 않습니다. (다만, 하위 호환성을 위해 사용은 가능) v2.6+ 부터 v-slot 디렉티브를 활용해 디렉티브 전달인자로 슬롯 이름을 설정합니다.




 





 



<app-card>

  <!-- header 이름을 가진 슬롯 -->
  <template v-slot:header> ... </template>

  <!-- 기본 슬롯 -->
  ...

  <!-- footer 이름을 가진 슬롯 -->
  <template v-slot:footer> ... </template>

</app-card>

오류 발생!

요소, 컴포넌트 외에서 사용할 경우 오류가 발생합니다.

[Vue warn]: Error compiling template:
v-slot can only be used on components or <template>.


기본 슬롯을 명시적으로 표기하고자 한다면 다음과 같이 코드를 작성할 수 있습니다.







 






<app-card>

  <!-- header 이름을 가진 슬롯 -->
  <template v-slot:header> ... </template>

  <!-- 기본 슬롯 -->
  <template v-slot:default> ... </template>

  <!-- footer 이름을 가진 슬롯 -->
  <template v-slot:footer> ... </template>

</app-card>

# 컴포넌트 컴파일 영역

컴포넌트 템플릿 영역의 모든 콘텐츠는 컴포넌트 자신 안에서만 컴파일 됩니다. 즉, 부모 컴포넌트 템플릿 영역의 모든 콘텐츠는 부모 컴포넌트 내에서만 컴파일 됩니다. 마찬가지로 자식 컴포넌트 또한 자신의 템플릿 영역 내에서 콘텐츠가 컴파일 됩니다.

예를 들어 컴포넌트 슬롯에 끼워질 콘텐츠의 인터폴레이션은 자식 컴포넌트가 아니라 부모 컴포넌트 영역에서 컴파일 됩니다.





 








 
 
 





<template>
  <div class="slot-compilation-scope-demo">
    <app-card>
      <!-- app-card가 아닌, 부모 컴포넌트 데이터 사용 -->
      <p>{{ user.name }}에게 후원해주세요.</p>
    </app-card>
  </div>
</template>

<script>
export default {
  data() {
    return {
      user: {
        name: '지호'
      }
    }
  }
}
</script>

# 슬롯 영역의 데이터 공유

앞서 다뤘듯이 슬롯이 컴포넌트 템플릿 재사용성을 높이기 위한 기능이지만, 슬롯에 삽입되는 콘텐츠는 외부 컴포넌트 템플릿 영역에서 컴파일 되므로 컴포넌트 데이터를 재사용할 수 없습니다.

# slot-scope 속성

하지만 slot-scope 속성을 사용하면 자식 컴포넌트의 데이터를 부모 컴포넌트 템플릿 영역에서 사용할 수 있도록 합니다.


사용법을 간단하게 알아보면 자식 컴포넌트 슬롯에서 부모 템플릿 영역으로 데이터를 전달하도록 설정 합니다.

자식 컴포넌트



 


<li v-for="phone in phones" :key="phone.id">
  <!-- 컴포넌트 외부로 phone 데이터 전달 -->
  <slot :phone="phone" />
</li>

자식 컴포넌트를 사용하는 부모 컴포넌트 템플릿 영역의 코드는 아래와 같이 <template> 요소로 끼워넣어질 콘텐츠를 감싼 다음 slot-scope 속성을 추가하여 자식 컴포넌트의 슬롯에서 전달된 데이터를 포함한 객체를 전달 받을 수 있습니다.

부모 컴포넌트



 



<app-phoneList :list="phoneList">
  <template slot-scope="scopeProps">
    {{ scopeProps.phone.name }}
  </template>
</app-phoneList>

# v-slot 디렉티브로 사용법 통일!

앞서 이야기 했듯이 v2.6+ 부터는 v-slot 디렉티브 사용을 권장합니다. v-slot 디렉티브 속성이 새롭게 추가되면서 슬롯 사용법이 일관되고 편리해졌습니다. 더 이상 slot 속성, slot-scope 속성과 같이 따로 구부하지 않아도 됩니다.

# 이름이 있는 슬롯

v-slot:[슬롯 이름]


 




<child-component>
  <template v-slot:header>
    <!-- 끼워넣어질 콘텐츠 -->
  </template>
</child-component>

# 슬롯 별칭 # 활용

Vue.js는 v-bind 디렉티브의 별칭 :, v-on 디렉티브의 별칭 @을 제공합니다. 마찬가지로 새롭게 추가된 v-slot 디렉티브 또한 별칭 #을 제공합니다. 이를 활용하면 구문을 보다 간결하게 줄일 수 있습니다.

#[슬롯 이름]="{ 데이터 }"


 







<child-component>
  <template #header="{ headline }">
    {{ headline.news }}
  </template>
  <template #default>
    <!-- 끼워넣어질 콘텐츠 -->
  </template>
</child-component>
