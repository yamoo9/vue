# Vue CLI

Vue CLI는 Vue.js 팀에서 제공하는 공식 개발도구로, Vue.js 싱글 파일 컴포넌트(SFC)와 ECMAScript 2015+를 프로젝트에 바로 사용할 수 있도록 도와줍니다. 프로젝트 구성 시 필요한 도구들(예: Babel, TypeScript, ESLint, Sass, Pug, PWA 등)을 선택하여 스캐폴딩 할 수 있습니다.

# 설치

Yarn 또는 NPM 패키지 매니저를 통해 @vue/cli 패키지를 설치합니다.

$ yarn global add @vue/cli  # 또는 npm i -g @vue/cli
1

NOTE

이전 버전인 vue-cli(1.x, 2.x)가 설치된 경우, 먼저 제거합니다. 권장되는 Node.js 버전은 8.11.0+ 입니다

# 설치 확인

설치가 완료되면 vue 바이너리를 CLI에서 사용할 수 있습니다. 설치된 @vue/cli 버전(3.x) 정보를 출력해봅니다.

$ vue --version  # 또는 vue -V
1

# 생성

create 명령을 실행해 프로젝트를 생성할 수 있습니다.

$ vue create <프로젝트_이름>
1

주의!

minTTY와 함께 Git Bash를 사용하는 Windows 사용자는 create 명령이 정상적으로 실행되지 안될 수 있습니다. 문제 해결을 위해 .bashrc 파일에 별칭(alias) 코드를 추가합니다.

# ~/.bashrc
alias vue='winpty vue.cmd'
1
2

※ 업데이트 된 .bashrc 파일을 가져 오려면 Git Bash 터미널 세션을 다시 시작해야 합니다.

# 구조

생성된 프로젝트 구조는 다음과 같이 정적 리소스와 프로젝트 리소스로 구분됩니다.

vue-cli 구 버전 활용

Vue CLI 3 버전인 @vue/cli에서도 구 버전인 vue-cli 명령을 사용할 수 있습니다. 구 버전을 사용해 프로젝트를 구성하려면 @vue/cli-init 패키지를 글로벌 설치해야 합니다.

$ yarn global add @vue/cli-init # 또는 npm i -g @vue/cli-init
1

vue-cli를 사용해 프로젝트를 구성하는 명령은 다음과 같습니다.

# vue init <프리셋_이름> <프로젝트_이름>
$ vue init yamoo9/vue-simple <프로젝트_이름>
1

# 원격 프리셋

프리셋을 Github 저장소에 올려 공유할 수 있습니다. 프리셋 공유에 필요한 파일은 다음과 같습니다.

  • preset.json: 사전 설정된 데이터가 포함된 메인 파일 (필수)
  • generator.js: 프로젝트에서 파일을 삽입하거나 수정할 수 있는 생성기
  • prompts.js: 옵션을 수집 할 수 있는 프롬프트 파일

공유된 프리셋을 사용하는 명령어 사용법은 아래와 같습니다.

# <프리셋_이름> <프로젝트_이름>
$ vue create --preset yamoo9/vue-ts <프로젝트_이름>
1
2

# 로컬 프리셋

원격 프리셋 개발 과정에서 매번 원격 저장소에 푸시 테스트하는 것은 지루하고 힘듭니다. 이를 간소화하기 위해 로컬 프리셋 설정을 사용할 수 있습니다. --preset 옵션 값이 절대 경로 또는 상대 경로일 경우 Vue CLI는 설정을 로드해 프로젝트를 생성합니다.

# 방법 1) ./<사용자_정의_프리셋> 디렉토리는 내부에 preset.json 파일을 포함해야 합니다.
$ vue create --preset ./<사용자_정의_프리셋> <프로젝트_이름>

# 방법 2) JSON 파일을 사용자 정의 프리셋으로 사용할 수도 있습니다.
$ vue create --preset <사용자_정의_프리셋>.json <프로젝트_이름>

# 플러그인

Vue CLI는 플러그인 기반 아키텍처를 사용합니다. 플러그인 기반 아키텍처는 Vue CLI를 유연하고 확장 가능하게 만듭니다. 플러그인 개발에 관심이 있다면 플러그인 개발 안내서를 참고하세요. Vue CLI에서 공식 제공하는 플러그인은 다음과 같습니다.

  • cli-plugin-babel
  • cli-plugin-eslint
  • cli-plugin-typescript
  • cli-plugin-pwa
  • cli-plugin-unit-jest
  • cli-plugin-unit-mocha
  • cli-plugin-e2e-cypress
  • cli-plugin-e2e-nightwatch

Vue CLI가 제공하는 플러그인을 프로젝트에 추가하려면 add 명령을 사용합니다.

$ vue add <플러그인_이름>
1

예를 들어 아래 명령으로 cli-plugin-eslint 플러그인을 프로젝트에 추가할 수 있습니다.

$ vue add @vue/cli-plugin-eslint

# 아래 명령은 위 명령과 동일합니다.
$ vue add @vue/eslint
1
2
3
4

Vue Router, Vuex는 Vue CLI의 플러그인은 아니지만, 특별히 add 명령으로 프로젝트에 추가할 수 있습니다.

$ vue add router
$ vue add vuex
1

# CLI Service

Vue CLI를 설치하면 vue-cli-service 바이너리 파일을 CLI 명령어 환경에서 실행할 수 있습니다.

$ npx vue-cli-service serve # 개발 서버 구동
$ npx vue-cli-service build # 배포를 위한 빌드
1
2

NOTE

npx는 최신 버전의 NPM을 사용할 경우 기본 설치 됩니다.

기본 프리셋으로 생성된 프로젝트의 package.json의 NPM 스크립트를 살펴보면 vue-cli-service 바이너리가 스크립트 명령에 사용된 것을 확인할 수 있습니다.

{
  "scripts": {
    "serve": "vue-cli-service serve",
    "build": "vue-cli-service build"
  }
}

$ yarn serve # 또는 npm run serve

# serve 명령

vue-cli-service serve 명령은 webpack-dev-server를 사용해 개발 서버를 구동합니다. (HMR 반영)

 

 
 
 
 
 
 
 
 

사용법: vue-cli-service serve [옵션] [entry]

옵션:

  --open    개발용 서버 URL로 브라우저로 열기
  --copy    개발용 서버 URL 클립보드에 저장
  --mode    env 모드 설정 (기본 값: development)
  --host    host 설정 (기본 값: 0.0.0.0)
  --port    port 설정 (기본 값: 8080)
  --https   https 사용 (기본 값: false)

# build 명령

vue-cli-service build 명령은 HTML, CSS, JavaScript 코드를 압축하고, 보다 ​​나은 성능(캐싱)을 위해 자동으로 분할된 청크(Chunk) 파일을 dist 디렉토리에 빌드합니다.

 

 
 
 
 
 
 
 
 
 
 
 
 

사용법: vue-cli-service build [옵션] [entry|pattern]

옵션:

  --mode        env 모드 설정 (기본 값: production)
  --dest        아웃풋 디렉토리 설정 (기본 값: dist)
  --modern      모던 브라우저를 대상으로 하는 자동 대체(fallback)가 반영된 앱 빌드
  --target      app | lib | wc | wc-async (기본 값: app)
  --formats     라이브러리 빌드 아웃풋 포멧 리스트 (기본 값: commonjs,umd,umd-min)
  --name        lib 또는 web-component 모드의 이름 (기본 값: package.json의 "name" 또는 엔트리 파일 이름)
  --no-clean    프로젝트를 빌드하기 전에 dist 디렉토리를 제거 안함
  --report      번들(묶음) 결과를 분석하는데 도움이 되는 report.html 생성
  --report-json 번들 콘텐츠를 분석하는데 도움이 되는 report.json 생성
  --watch       변경사항 관찰

옵션 중 몇가지는 매우 유용합니다.

  • --modern : 모던 모드를 사용해 앱을 빌드 (성능 개선, 파일 크기 감소)
  • --target : 라이브러리 또는 웹 컴포넌트로 프로젝트에 사용된 Vue 컴포넌트를 작성할 수 있습니다. (빌드 타겟 참고)
  • --report 또는 --report-json : 번들에 포함된 모듈의 크기를 분석 할 수 있는 빌드 통계를 기반으로 보고서를 생성합니다.

사용 예시


 


 


 

# ECMAScript 2015를 지원하는 모던 브라우저 용 빌드
$ vue-cli-service build --modern

# ※ 참고로 웹 컴포넌트 모드는 IE를 지원하지 않습니다.
$ vue-cli-service build --target wc --name y9 'src/components/*.vue'

# 리포트 결과를 report.html 파일로 생성합니다.
$ vue-cli-service build --report

# 환경설정

기본 환경설정을 변경하거나 사용자가 임의로 설정하고 싶다면, 프로젝트 루트에 vue.config.js 파일을 만든 후 Vue CLI 설정을 참고해 설정을 추가하거나 변경하면 vue-cli-service를 통해 반영됩니다. 예를 들어 빌드 디렉토리의 기본 이름인 dist 대신, build를 사용하고자 설정한다면 outputDir 속성 값을 변경합니다.





 


// vue.config.js

export.defaults = {
  // 설정
  outputDir: 'build'
};

// vue.config.js

      new MyAwesomeWebpackPlugin()

# CSS 프리 프로세서

Vue CLI 프로젝트는 PostCSS, CSS 모듈 및 Sass, Less, Stylus를 포함한 프리 프로세서를 지원합니다. 프로젝트 생성 과정에서 프리 프로세서를 설정하지 않을 경우, Webpack 로더를 추가적으로 설치하면 자동으로 프로젝트에 사용할 수 있습니다.

예를 들어 Sass 프리 프로세서를 프로젝트에 사용하고자 한다면 node-sass, sass-loader 패키지를 설치합니다.

$ yarn add -D node-sass sass-loader # 또는 npm i -D node-sass sass-loader
1

패키지를 설치한 후에는 Vue 컴포넌트 스타일 코드에서 프리 프로세서를 활용할 수 있습니다.

 







<style lang="scss">
$textColor: #292929;

body {
  color: $textColor;
}
</style>
1
2
3
4
5
6
7

Sass 프리프로세서에 옵션을 설정하려면 css.loaderOptions에 설정할 수 있습니다. 예를 들어 Sass 글로벌 변수를 모든 Sass 파일(scss, sass)에 전달할 수 있습니다.

// vue.config.js

module.exports = {
  css: {
    loaderOptions: {
      sass: {
        data: `@import "@/variables.scss";`
      }
    }
  }
};
