# 관찰(Watch) 속성

# 관찰 속성이 필요한 경우

일반적으로 계산된 속성을 주로 사용하지만, 특별한 경우 관찰 속성이 필요한 경우가 있습니다. 데이터 변경에 많은 시간이 소요되거나, 비동기 통신의 응답 결과를 기다려야 할 때 관찰 속성을 사용해야 합니다. 반면 간단한 연산 식을 처리하는 용도는 계산된 속성을 사용합니다. 즉, 동기(sync) 처리는 계산된 속성을, 비동기(async) 처리는 관찰 속성을 사용한다고 기억하세요.

# 데이터 지속 관찰

관찰 속성 사용법은 다음과 같이 관찰할 데이터 속성과 동일한 이름을 watch 객체의 속성으로 설정합니다. 속성은 함수 형태이고 첫번째 매개변수는 현재 값(value)을, 두번째 매개변수는 이전 값(oldValue)을 전달 받습니다. created 훅 함수 내부에 설정된 1초마다 page 데이터 속성 값을 +2 만큼 업데이트 하면 관찰 속성은 이를 감지하여 현재 값과 이전 값을 콘솔 패널에 출력합니다.





 
 
 
 
 
 
 
 
 







export default {
  data: () => ({
    page: 1
  }),
  watch: {
    page(value, oldValue) {
      console.log(`value=${value}, oldValue=${oldValue}`);
      // value=3, oldValue=1
      // value=5, oldValue=3
      // value=7, oldValue=5
      // value=9, oldValue=7
      // ...
    }
  },
  created() {
    // 1초 마다 page 데이터 속성 값 변경
    window.setInterval(() => (this.page += 2), 1000);
  }
}

# 비동기 응답 처리

랜덤 사용자 생성 온라인 서비스 randomuser.me를 이용해 비동기 통신의 응답 결과를 관찰 속성으로 처리해보겠습니다.

# 사용법

randomuser.me API를 참고하여 페이지, 결과(개수)를 서버에 요청하도록 설정합니다. fetch API는 Promise를 반환하기에 .then() 메서드를 사용해 비동기 응답 받은 결과를 처리할 수 있습니다.

let [page, limit] = [1, 5];

fetch(`https://randomuser.me/api/?page=${page}&results=${limit}`)
  .then(response => response.json())
  .then(({results}) => console.log(results));

# 컴포넌트 설정

서버에 요청할 때 사용할 page, limit, 비동기 응답 결과를 푸시할 users 속성을 데이터에 추가한 후, 초기 값을 설정합니다.

data: () => ({
  users: [],
  page: 1,
  limit: 5
}),

이어서 서버에서 데이터를 패치(fetch)하는 fetchData 메서드를 작성합니다. 비동기 응답 받은 데이터는 복사하여 users 배열에 푸시하여 users 데이터를 업데이트 합니다.

methods: {
  fetchData() {
    const { page, limit } = this;
    fetch(`https://randomuser.me/api/?nat=US&page=${page}&results=${limit}`)
      .then(response => response.json())
      .then(({results}) => {
        this.users.push(...results);
      });
  }
}

# 관찰 처리

page 데이터를 지속적으로 관찰하는 속성을 추가한 후, 데이터가 변경되면 서버로부터 데이터를 패치합니다.

watch: {
  page(value) {
    this.fetchData();
  }
}

관찰 속성 프로그래밍이 완료되었습니다. 이제 사용자 입장에서 "더 많은 사용자 불러오기" 버튼을 누르면, page 데이터 값이 변화되고, 이를 관찰하는 속성 page()에 의해 서버에 변경된 페이지 값을 전달하여 서버에 데이터를 요청하여 가져옵니다. 최종적으로 가져온 데이터는 users를 업데이트 하게 되고, 사용자 UI는 업데이트 됩니다. 아래 예제를 살펴보세요.


NOTE

페이크 온라인 REST API jsonplaceholder와 Promise 기반 HTTP 클라이언트 라이브러리 axios를 사용해 비동기 응답 처리 테스트를 진행할 수도 있습니다.

import axios from 'axios';

let [resource, page, limit] = ['albums', 1, 10];
let api_address = `https://jsonplaceholder.typicode.com/${resource}`;

axios.get(`${api_address}?_page=${page}&_limit=${limit}`)
  .then(response => console.log(response))
  .catch(error => console.error(error));

# 관찰 즉시 실행

일반적으로 관찰 속성은 초기 로딩 과정에서 실행되는 것이 아니라, 로딩 이후 관찰 결과 데이터가 변경되면 실행됩니다. 기존에 작성했던 created 훅 함수는 제거하고, page 관찰 속성을 아래와 같이 handler와 immediate로 구분해 작성합니다. 그리고 immediate 값을 true로 설정하면 초기 로딩 이후 즉시 관찰을 실행해 결과를 반영합니다.










 



// created() {
//   this.fetchData();
// },

watch: {
  page: {
    handler(value, oldValue) {
      this.fetchData();
    },
    immediate: true
  }
}

# 깊은 관찰

객체를 포함하는 상위 객체를 관찰할 때, 하위 객체까지 관찰하는 깊은 관찰을 설정할 경우 deep 속성을 설정합니다. 속성이 설정되면 관찰 중인 객체의 깊은 곳에서의 변화까지 관찰 감지할 수 있습니다.



 
 
 
 
 
 
 








 



data() {
  return {
    book: {
      title: 'Vue.js 러닝 가이드북',
      toc: [
        { chapterNo: 1, title: 'Vue.js란?' },
        // ...
      ]
    }
  }
},
watch: {
  book: {
    handler(value, oldValue) {
      console.log('객체가 포함하는 하위 객체까지 깊은 관찰');
      console.dir(value);
    },
    deep: true
  }
}

하지만 객체가 복잡할 경우, 모든 것을 관찰하는 것은 성능 문제를 발생시킬 수 있습니다. 이런 경우 객체의 모든 하위 객체를 관찰하기 보다는 특정 하위 객체를 관찰하는 것이 좋습니다.


 





watch: {
  'book.toc': function(value, oldValue) {
    console.log('객체가 포함하는 특정 하위 객체만 관찰');
    console.dir(value);
  }
}

# $watch

watch 속성 대신 $watch 메서드를 사용해 데이터 속성을 관찰 할 수 있습니다.

created() {
  this.$watch('book', (value, oldValue) => {
    console.log('관찰 속성 book:', value);
  });
}

# 관찰 중지

관찰 속성은 성능에 영향을 미치므로 관찰이 더 이상 필요하지 않으면 멈출 수 있도록 할 수 있습니다. $watch 메서드는 관찰을 멈출 수 있는 함수를 반환합니다. 이 메서드를 참조한 후, 관찰을 중지할 때 메서드를 실행하면 관찰은 중지됩니다.


 





 
 
 


created() {
  this.watchBookId = this.$watch('book', this.watchBook);
}
methods() {
  watchBook(value, oldValue) {
    console.log('관찰 속성 book:', value);
  },
  unWatchBook() {
    this.watchBookId();
  }
}
