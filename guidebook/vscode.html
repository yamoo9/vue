# 에디터 테마

# Vue Theme

VSCode 에디터의 테마로 Vue Theme 설치를 권장합니다. (다른 테마를 설치해도 무방합니다.)

# Material Icon Theme

VSCode 에디터 아이콘 테마로 Material Icon Theme 설치를 권장합니다. (다른 아이콘 테마를 설치해도 무방합니다.)



# Vue 개발을 위한 확장

# Vetur

Vetur 확장은 .vue 파일 구문 강조, Emmet, 린팅/오류 검토, 포멧팅, 자동 완성, 디버깅을 포함합니다.

# Vue Peek

vue-peek 확장은 .vue 파일 코드를 열지 않고 엿보거나(peek), 이동 할 수 있는 기능을 제공합니다.

# ESLint

ESLint 확장은 ECMAScript 문법 검사를 수행하고, 지정된 포멧에 맞춰 자동 수정할 수 있습니다.

NOTE

ESLint 패키지를 설치한 적이 없다면 eslint 패키지를 설치해야 합니다.

 


$ yarn global add eslint
# npm i -g eslint
1
2

# Prettier

Prettier 코드 포멧 확장은 일관성 있는 코드를 유지할 수 있도록 자동으로 지정된 포멧으로 수정합니다.

# Formatting Toggle

Formatting Toggle 확장을 사용하면 자동으로 코드를 수정하는 코드 포멧 확장을 활성/비활성 토글할 수 있습니다.

# NPM Intellisense

npm Intellisense 확장은 NPM 모듈을 import 하는 구문을 자동 완성할 수 있게 합니다.

# Path Intellisense

Path Intellisense 확장은 파일을 빠르게 찾아 입력할 수 있도록 도와줍니다.

주의!

Windows 환경에서는 자동 제안 이슈를 참고하여 . 입력 시 제안되는 바인딩을 해제합니다.

// keybindings.json

{ "key": ".", "command": "" }

# JavaScript (ES6) code snippets

JavaScript (ES6) code snippets 확장은 ES6 코드 스니펫으로, 확장을 설치하면 ES6 코드를 빠르게 작성할 수 있습니다.

# Import Cost VSCode

Import Cost VSCode 확장은 import/require 되는 패키지의 크기를 에디터에 표시해줍니다.

# Bracket Pair Colorizer

Bracket Pair Colorizer 확장은 매칭되는 브라켓에 색상을 설정해 코드를 읽기 용이하도록 만들어줍니다.

# Sorting HTML and Pug attributes

Sorting HTML and Pug attributes 확장은 설정된 순서대로 HTML 또는 Pug 속성을 자동으로 정렬합니다.

# Auto Rename Tag

Auto Rename Tag 확장은 자동으로 HTML/XML 태그 이름을 수정합니다.

# Auto Close Tag

Auto Rename Tag 확장은 자동으로 HTML/XML 닫는 태그를 추가해줍니다.



# ESLint 설정

설정 파일을 개별 분리 관리할 경우, 프로젝트 루트 위치에 .eslintrc.js 파일을 만들어 다음과 같이 설정합니다.







 



 








// .eslintrc.js

module.exports = {
  root: true,
  env: {
    node: true,
    browser: true
  },
  extends: [
    "plugin:vue/essential",
    "plugin:prettier/recommended",
    "@vue/prettier"
  ],
  "rules": {},
  "parserOptions": {
    "parser": "babel-eslint"
  }
}

# Prettier 설정

프로젝트 루트 위치에 .prettierrc.js 파일을 만들어 아래와 같이 설정합니다. (옵션 참고)

// .prettierrc.js

module.exports = {
  // 줄 바꿈 길이 설정
  printWidth: 80,
  // 탭(true), 스페이스(false) 사용 설정
  useTabs: false,
  // 탭(tab) 공백 수 설정
  tabWidth: 2,
  // 작은 따옴표(') 사용 설정
  singleQuote: true,
  // 명령문 끝에 세미콜론 추가 설정
  semi: true,
  // 화살표 함수 => 단일 매개변수 괄호 () 설정
  arrowParens: "always",
  // 객체 또는 배열의 여러 줄 입력 시, 후행 쉼표 추가 설정
  trailingComma: "es5",
  // HTML 공백 감도 설정
  htmlWhitespaceSensitivity: "ignore",
  // 객체 리터럴 괄호({ }) 사이 공백 출력
  bracketSpacing: true
}

# 에디터 설정

Vue 애플리케이션 개발 최적화를 위한 설정을 settings.json 파일에 추가합니다.

settings.json 파일 열기

  1. VS Code에서 F1 키를 누른 후, user settings를 검색 실행합니다.
  2. Settings 설정 검색에 code를 입력한 후, settings.json 파일에서 편집을 클릭합니다.
  3. 사용자 설정 패널에 아래 코드를 복사 붙여넣기 한 후, 저장합니다.
// settings.json

// 기본 JavaScript 포멧 비활성화
"javascript.format.enable": false,

// ESLint + Prettier 사용을 위해 Vetur 린팅 기능 끄기
"vetur.validation.template": false,

// ESLint 검사 설정:  *.{html,js,vue}
// - 검사 후 문제 자동 수정
"eslint.validate": [
  {
    "autoFix": true,
    "language": "html"
  },
  {
    "autoFix": true,
    "language": "javascript"
  },
  {
    "autoFix": true,
    "language": "vue"
  },
  {
    "autoFix": true,
    "language": "javascriptreact"
  }
],
// ESLint 사용 설정
"eslint.enable": true,
// ESLint 자동 수정 활성화
"eslint.autoFixOnSave": true,
// true일 경우, 파일 저장 시 자동 포멧 실행
"editor.formatOnSave": true,

// 에디터 설정
"editor.tabSize": 2,
"editor.formatOnSave": false,
"editor.renderWhitespace": "all",

// 포멧 수정을 토글(on/off)하는 설정: 저장할 때
// ※ Formatting Toggle 확장 설치한 경우만
"formattingToggle.activateFor": ["formatOnSave"],

// 속성 정렬(attrsSorter) 설정
// ※ Sorting HTML and Jade attributes 확장 설치한 경우만
"attrsSorter.order": [
  "is",
  "v-for",
  "v-if",
  "v-else-if",
  "v-else",
  "v-show",
  "v-cloak",
  "v-once",
  "v-pre",
  "id",
  "ref",
  "key",
  "slot",
  "v-model",
  "v-model.+",
  "v-bind",
  "v-bind.+",
  ":.+",
  "v-text",
  "v-text.+",
  "v-html",
  "v-html.+",
  "class",
  "v-on.+",
  "@.+",
  "name",
  "data-.+",
  "ng-.+",
  "src",
  "for",
  "type",
  "href",
  "values",
  "title",
  "alt",
  "role",
  "aria-.+",
  "$unknown$"
],

# 코드 스니펫

snippet-generator 온라인 도구를 사용해 스니펫 코드를 쉽게 생성합니다.

# 야무의 스니펫

Vue.js 개발을 편하게 만들어주는 야무의 스니펫을 다운로드하여 사용할 수 있습니다.


# 스니펫 저장 위치

# Mac

/Users/<사용자계정이름>/Library/Application Support/Code/User/snippets
1

# Windows

C:\Users\<사용자계정이름>\AppData\Roaming\Code\User\snippets
1

NOTE

사용자 정의 스니펫을 사용할 경우, VSCode 옵션에서 Vetur 스니펫을 사용하지 않도록 설정합니다.

// settings.json

"vetur.completion.useScaffoldSnippets": false
