# 의존성 주입

# $parent 속성 사용의 문제점

Google Map 예제에서 $parent 속성을 사용해 자식 컴포넌트가 부모 컴포넌트의 메서드에 접근 사용하는 방법을 살펴봤습니다.

<google-map>
  <google-map-marker :places="cities"/>
</google-map>
1
2
3

잘 작동되는 코드지만 아래와 같이 마크업이 변경되면 문제가 발생합니다. google-map-marker 자식 컴포넌트가 참조하는 $parent 대상이 변경되었기 때문입니다.


 

 


<google-map>
  <google-map-region :shape="cityBoundaries">
    <google-map-marker :places="cities"/>
  </google-map-region>
</google-map>

이런 경우 자식 컴포넌트의 코드를 다음과 같이 수정해야 합니다.


 



created() {
  const getMap = this.$parent.getMap || this.$parent.$parent.getMap;
  getMap(map => { ... });
}

주의!

부모 컴포넌트가 수시로 변동될 가능성이 농후한 경우, $parent 속성을 사용하는 것은 유지보수 하기 어렵습니다.



# 해결책 : 공급/주입

앞서 거론한 문제를 해결하기 위한 의존성 주입(Dependency Injection) 방법으로 provide, inject 속성을 제공합니다. 이 옵션 쌍은 함께 사용하여 상위 컴포넌트가 컴포넌트 계층 구조 깊이에 관계없이 모든 하위 항목에 대한 종속성을 주입하는 역할을 하도록 허용합니다.

# 공급(Provide)

부모 컴포넌트는 provide 속성(함수)을 통해 자식 컴포넌트에 데이터 및 메서드를 공급할 수 있습니다.

GoogleMap.vue

export default {
  provide() {
    return {
      getMap: this.getMap
    }
  }
}

# 주입(Inject)

자식 컴포넌트는 inject 속성을 통해 부모 컴포넌트가 주입한 데이터 및 메서드를 주입 받을 수 있습니다.

GoogleMapMarkder.vue

export default {
  inject: ['getMap']
}

주의!

provide와 inject 속성은 주로 고급 플러그인/컴포넌트 라이브러리를 위해 제공됩니다. 일반 애플리케이션 코드에서는 사용하지 않는 것이 좋습니다.
