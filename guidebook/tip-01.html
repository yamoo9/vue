# 빅 데이터 관리

일반적으로 애플리케이션은 초기 로딩 시에 데이터(예: 배열, 객체 등)를 가져와야 사용합니다. 가져온 데이터는 Vue.js의 반응성 시스템에 의해 관리됩니다. 문제는 반응성 시스템에 의해 빅 데이터(예: 관라하기 큰 객체)를 관리할 경우 적지 않은 비용이 듭니다.

export default {
  data() {
    return {
      users: {}
    }
  },
  async created() {
    const users = await axios.get('/api/user');
    this.users = users;
  }
}

이런 경우 컴포넌트에 데이터를 추가하기 전에 Object.freeze를 사용해 관리할 데이터를 동결(freeze)한 다음 컴포넌트 데이터로 할당하면 성능 향상에 도움이 됩니다.









 



export default {
  data() {
    return {
      users: {}
    }
  },
  async created() {
    const users = await axios.get('/api/user');
    this.users = Object.freeze(users);
  }
}

이 방법은 Vuex에서도 동일하게 적용할 수 있습니다.

mutations: {
  fetchUsers(state, users) {
    state.users = Object.freeze(users);
  }
}

하지만 데이터를 동결하면 데이터를 수정할 수 없어 문제가 됩니다. 동결된 데이터를 수정하려면 다음과 같이 데이터를 복제한 후, 새로운 아이템을 추가하는 방법을 사용합니다.



 



mutations: {
  addUser(state, user) {
    state.users = Object.freeze([...state.users, user]);
  }
}

NOTE

중첩된 객체를 포함하는 복잡한 객체를 복제할 경우 깊은 복사를 수행하는 deepCopy 유틸리티 함수를 사용할 수 있습니다.

const deepCopy = o => JSON.parse(JSON.stringify(o));
1

유틸리티 deepCopy를 활용하면 복잡한 객체를 깊은 복사할 수 있습니다.

state.users = Object.freeze([...deepCopy(state.users), user]);
