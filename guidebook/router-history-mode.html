# URL에서 #/ 제거

Vue Router 라이브러리는 기본적으로 해시 모드(Hash Mode)로 동작합니다. 이 모드는 URL에 #/를 삽입해 서버로 요청하지 않고, 클라이언트에서 URL을 해석하여 라우팅 할 수 있도록 만들어 줍니다. 해시 모드 대신 HTML5 히스토리 모드를 사용하려면 라우터에 mode 옵션 값을 history로 설정합니다.




 



// router.js

export default new Router({
  mode: 'history',
  routes
});

# 서버 설정

히스토리 모드를 사용하면 #/이 URL에서 제거되어 서버 사이드 렌더링 한 것처럼 URL이 표시되지만, 싱글 페이지 애플리케이션의 특성 상 사용자가 직접 URL을 입력(예: https://service.io/course/vue) 하거나 즐겨찾기 한 경우, 정상적으로 페이지를 렌더링하지 못합니다.

이 문제는 서버 설정으로 쉽게 해결할 수 있습니다. 이 설정은 URL이 일치하지 않아 오류가 발생할 경우, 애플리케이션의 index.html 페이지를 대체 제공하도록 하는 것입니다. 서버 유형마다 설정 방법이 다르니 서비스하는 서버 유형에 맞는 방법을 사용해야 합니다.

Apache

<IfModule mod_rewrite.c>
  RewriteEngine On
  RewriteBase /
  RewriteRule ^index\.html$ - [L]
  RewriteCond %{REQUEST_FILENAME} !-f
  RewriteCond %{REQUEST_FILENAME} !-d
  RewriteRule . /index.html [L]
</IfModule>

# 주의할 점

서버는 페이지를 찾을 수 없는(404) 오류를 보고하지 않습니다. 서버 설정에 따라 찾을 수 없는 URL일 경우, index.html로 대체 연결하기 때문입니다. 문제 해결을 위해 와일드 카드(*) URL에 PageNotFound 컴포넌트를 연결하도록 설정합니다. 이 설정으로 404 오류가 발생하면 PageNotFound 컴포넌트가 화면에 표시됩니다.







 



// router.js

export default new Router({
  mode: 'history',
  routes: [
    // ...
    path: '*', component: PageNotFound
  ]
})
