    # 성능 측정

    앞서 다룬 빅 데이터 관리 성능 향상 팁에서 큰 객체와 같은 데이터를 관리하는 효율적인 방법을 알아봤습니다. 하지만 그 팁을 사용한 방법이 얼마 만큼 성능을 개선했는지 체감되진 않습니다. 이번 팁은 Chrome 개발도구(DevTools)의 성능(performance) 탭을 사용해 성능을 측정하는 방법을 알아보는 것입니다.

    # 테스트 준비

    정확한 성능 측정 데이터를 얻으려면 Vue 애플리케이션에서 성능(performance) 모드를 활성화 해야 합니다.

    main.js

    Vue.config.performance = true;
    1

    NOTE

    Vue.js는 컴포넌트의 성능을 표시하기 위해 내부적으로 사용하는 User Timing API를 활성화 합니다.

    # 테스트

    성능 측정 테스트와 테스트 코드를 살펴보며 성능 비교를 해봅시다. Chrome 개발도구를 열어 성능 탭을 클릭한 후, 성능 탭의 Start profiling and reload page 버튼을 눌러 페이지를 새로고침 한 후 프로파일링을 시작합니다. 페이지 성능 테스트 결과가 기록되면, Timings 측정 부분을 확인합니다.


    # 측정 항목

    측정되는 항목은 총 3가지로 초기화, 렌더, 패치 순으로 진행됩니다.

    항목 설명
    초기화(init) 컴포넌트 인스턴스를 만드는 시간
    렌더(render) 가상 DOM 구조를 만드는 시간
    패치(fetch) 가상 Dom ⇒ 실제 DOM 적용 시간

    반응성 시스템을 사용하는 ReactiveTester 컴포넌트의 초기화 시간이 403.00ms인 반면, 반응성 시스템을 사용하지 않고 객체를 동결(freeze) 사용한 NonReactiveTester 컴포넌트의 초기화 시간은 15.36ms로 성능이 대폭 향상된 것을 확인할 수 있습니다.

    반응성 시스템의 데이터 초기화에 소요되는 시간


    객체를 동결/복사한 데이터 초기화에 소요되는 시간


    NOTE

    테스트 결과는 실행 시마다 시간이 다르게 측정되지만, 테스트에 사용된 컴포넌트 초기화 성능은 비교 확인 가능합니다.
