# 디렉티브(Directives)

디렉티브는 요소를 쉽게 조작하기 위한 문법을 말하며, '지시어' 또는 '지시문'으로 번역되고 있습니다. Vue.js는 UI 조작에 자주 사용되는 몇 가지 방법을 디렉티브로 제공하고 있습니다.

# 전달인자(arguments) / 수식어(modifier)

디렉티브는 필요한 경우, 전달인자 또는 수식어를 설정할 수 있습니다.

<요소 v-디렉티브이름:전달인자.수식어="값"></요소>
1

# v-text

v-text 디렉티브는 요소의 textContent 를 업데이트 합니다. 일부 텍스트만 업데이트할 경우 인터폴레이션을 사용하세요.

<!-- 전체 텍스트 데이터 바인딩 -->

# v-if

Vue.js가 제공하는 조건부 렌더링 디렉티브는 3가지로 사용법은 JavaScript 문법과 유사합니다.

  • v-if 디렉티브는 조건이 참이면 화면에 렌더링합니다.
  • v-else 디렉티브는 v-if 또는 v-else-if 뒤에 배치되어야 하며, 'else if' 블록입니다.
  • v-else-if 디렉티브는 v-if 또는 v-else-if 뒤에 배치되어야 하며, 'else' 블록입니다.
<div class="products-info">
  <p v-if="products.lenth > 1">제품 재고가 {{ products.length }}개 남아 있습니다.</p>
  <p v-else-if="products.length === 1">단 1개의 제품이 남아 있습니다.</p>
  <p v-else>제품 제고가 없습니다.</p>
</div>

<li v-for="book in books" v-if="book.isAvalableSale">

# v-for

v-for 디렉티브는 컴포넌트 데이터(예: 배열, 객체 등)를 순환 처리할 때 사용됩니다.

<tr v-for="book of books" v-bind:key="book.id">
  <td class="book-id">{{ book.id }}</td>
  <td class="book-name">{{ book.name }}</td>
  <td class="book-goods">{{ book.goods }}</td>
</tr>
1
2
3
4
5

그리고 인덱스(index)를 사용할 경우 아래와 같이 사용합니다.

<tr v-for="(book,index) of books" v-bind:key="index">
  <td class="book-id">{{ book.id }}</td>
  <td class="book-name">{{ book.name }}</td>
  <td class="book-goods">{{ book.goods }}</td>
</tr>
1
2
3
4
5

객체를 순환 처리할 경우에는 값(value), 속성(key), 인덱스(index)를 사용할 수 있습니다.

<tr v-for="(value, key, index) of book" v-bind:key="index">
  <th scope="row" class="book-key">{{ key }}</th>
  <td class="book-value">{{ value }}</td>
</tr>
1
2
3
4

NOTE

key 속성은 순환되는 각 아이템을 식별하는 고유 ID로 v-for 디렉티브 사용 시 함께 사용하는 것이 권장됩니다.

주의!

key 속성 값은 숫자 또는 문자와 같은 데이터 유형만 사용해야 합니다.

NOTE

v-if 디렉티브와 마찬가지로 <template> 요소를 사용해 여러 요소 블록을 리스트 렌더링 할 수 있습니다.

<ul>
  <template v-for="(item,v) of items" v-bind:key="v">
    <li>{{ item.title }}</li>
    <li>{{ item.msg }}</li>
    <li class="divider" role="presentation"></li>

# v-show

v-show 디렉티브는 v-if 디렉티브와 유사해보이지만, 몇 가지 차이점이 있습니다. <template>에 사용할 수 없고, v-else 디렉티브가 뒤에 이어질 수 없습니다. 그리고 조건부 렌더링이 아닌 CSS로 화면에서 감춤(display: none) 처리합니다.

<div v-show="mode === 'signup'">
  <!-- ... -->
</div>
<div v-show="mode !== 'signup'">
  <!-- ... -->
</div>
1
2
3
4
5
6

NOTE

사용자에 의해 수시로 보임/감춤 처리되는 토글 기능에 적합한 디렉티브는 v-show 입니다. v-if는 런타임 중에 조건이 변경되지 않는 조건부 렌더링에 적합한 디렉티브 입니다.

# v-bind

v-bind 디렉티브는 요소에 속성을 동적으로 바인딩할 때 사용합니다.

<!-- 속성 바인딩 -->
<img v-bind:src="image.src" v-bind:alt="image.alt">
1
2

v-bind 대신 : 별칭을 사용할 수 있습니다.

<!-- v-bind 별칭 -->
<img :src="image.src" :alt="image.alt">
1
2

JavaScript 식을 사용해 값을 처리할 수도 있습니다.

<!-- JavaScript 식 (문자 접합) -->
<img :src="`./images/${image.fileName}`" :alt="image.alt">
1
2

class, style 속성의 경우는 여러 값을 처리하기 때문에 특별히 객체 또는 배열을 값으로 설정해야 합니다.

<!-- class 속성 바인딩 -->
<div :class="{ point: isPoint }"></div>
<div :class="[classA, classB]"></div>
<div :class="[classA, { classB: isB, classC: isC }]">

<!-- style 속성 바인딩 -->
<div :style="{ marginTop: `${marginTopSize}px` }"></div>
<div :style="[styleObjA, styleObjB]"></div>

v-bind 디렉티브에 인자(arguments)를 전달하지 않고 아래와 같이 사용할 수도 있습니다.

<!-- 객체 속성 바인딩 -->
<div v-bind="{ id: bigHeader.id, title: bigHeader.title }"></div>
1
2

속성(attributes) 대신 DOM 속성(properties)을 바인딩 할 수 있습니다. 쉽게 말해 HTML 속성(attribute)이 아닌, JavaScript 객체 속성(property)을 설정한다는 말입니다. 설정된 속성은 JavaScript 객체의 속성이라 콘솔의 요소 패널을 통해 확인되지 않습니다. 즉, 표준 속성이 아닌 사용자 정의 속성을 설정할 때 유용하게 사용됩니다.

<!-- 속성 대신 DOM 속성 바인딩 -->
<div ref="el" v-bind:creator.prop="author"></div>

<!-- prop 바인딩. (my-component 컴포넌트에 props가 설정되어 있어야 함) -->
<my-component :foo="bar"></my-component>

<!-- 부모 컴포넌트의 데이터 속성을 모두 자식 컴포넌트의 props로 전달 -->
<child-component v-bind="$props"></child-component>

# v-on

v-on 디렉티브는 요소에 이벤트 리스너를 연결합니다. 이벤트 유형은 인자로 전달하며, 값은 메서드 이름 또는 인라인 문장, JavaScript 식을 사용할 수 있습니다.

<!-- 메서드 이름 -->
<button v-on:click="changeSignInMode">로그인 모드로 변경</button>

<!-- 메서드 이름(전달인자) -->
<button v-on:click="changeMode('signin', $event)">로그인 모드로 변경</button>
1
2
3
4
5

v-on 디렉티브 대신 별칭 @을 활용할 수 있습니다.

<!-- 별칭 @ -->
<button @click="changeSignInMode">로그인 모드로 변경</button>
1
2

v-on 디렉티브는 이벤트 수식어와 키 수식어를 제공합니다.

이벤트를 차단하거나, 기본 동작을 무시하려면 .stop, .prevent 수식어를 사용합니다.

<!-- 기본 동작 차단(prevent default) -->
<a href="/" @click.prevent="changePath('home')">홈</a>

<!-- 이벤트 전파 중지(stop propagation) -->
<a role="button" @click.stop="foldPanel">패널 접기</a>

<!-- 수식어 체이닝 -->
<button @click.stop.prevent="changeSignInMode">로그인 모드로 변경</button>

<!-- 폼 전송 기본 동작 차단 -->
<form @submit.prevent></form>

<!-- 객체 문법을 사용해 멀티 이벤트 연결 (2.4.0+) -->
<div v-on="{ mouseenter: playRotate, mouseleave: stopRotate }">...</div>

<!-- 이벤트 캡처링 사용 -->
<a href @click.capture="onCapturing">...</a>

<a href @click.prevent.self="">...</a>

# v-model

v-model 디렉티브는 사용자에게 입력 받은 데이터를 양방향 바인딩 하는 경우 사용합니다. 이 디렉티브는 HTML 폼 컨트롤 요소와 Vue 컴포넌트로 사용이 제한적입니다.

  • <input>
  • <select>
  • <textarea>
  • Vue 컴포넌트


 


<div class="form-control">
  <label for="email">이메일</label>
  <input id="email" v-model="email">
</div>

다음의 수식어와 함께 사용할 수 있습니다.

  • .lazy - input 이벤트 대신 change 이벤트 사용
  • .number - 입력된 문자를 숫자로 변환
  • .trim - 양쪽 공백을 제거(trim)


 




 



<div class="form-control">
  <label for="career-duration">경력기간</label>
  <input id="career-duration" v-model.number="career">
</div>

<div class="form-control">
  <label for="comment">코멘트</label>
  <textarea id="comment" v-model.lazy.trim="comment"></textarea>
</div>

# v-once

v-once 디렉티브는 데이터 바인딩을 1회만 설정합니다. 렌더링 이후 콘텐츠가 업데이트가 없을 경우 사용합니다.

<h2 v-once>{{ title }}</h2>

# v-cloak

v-cloak 디렉티브는 Vue 인스턴스가 준비 될 때까지 컴파일되지 않은 콧수염 데이터 바인딩을 숨기는 데 사용됩니다.

<!-- 전체 텍스트 데이터 바인딩 -->
<div id="app" v-cloak></div>
1
2

이 디렉티브는 아래 CSS 코드 설정이 필요합니다. [v-cloak] 속성을 가진 요소를 표시하지 않도록 만들어야 하기 때문입니다. 렌더링이 완료되면 Vue.js에서 v-cloak 속성을 제거해 요소를 화면에 표시하게 됩니다.

[v-cloak] { display: none }
1

# v-pre

v-pre 디렉티브가 설정된 요소(하위 요소 포함) 컴파일을 무시합니다. 별도로 제어할 필요가 없는 일반 HTML 요소에 설정하면 컴파일 속도를 개선할 수 있습니다.

<div v-pre id="will-not-be-compiled">Vue.js에서 제어할 필요가 없는 일반 HTML 영역</div>

# v-slot

v-slot 디렉티브는 슬롯(Slot)을 사용할 때 설정합니다. 기본(default) 슬롯, 이름이 부여된(named) 슬롯, 속성(props)을 전달하는 슬롯 등 v-slot 디렉티브를 사용해 처리할 수 있습니다. 자세한 내용은 v-slot 디렉티브로 사용법 통일을 참고하세요.

<!-- 이름이 부여된 슬롯(Named slots) -->
<base-layout>
  <template v-slot:header>
    헤더 콘텐츠
  </template>

  기본 슬롯 콘텐츠

  <template v-slot:footer>
    푸터 콘텐츠
  </template>
</base-layout>

<!-- 속성(props)을 전달 받는 이름이 부여된 슬롯 -->
<infinite-scroll>
  <template v-slot:item="slotProps">
    <div class="item">
      <!-- slotProps 객체의 속성(데이터) 활용 -->
      {{ slotProps.item.text }}
    </div>
  </template>
</infinite-scroll>

<!--
  속성(props)을 전달 받는 이름이 부여된 슬롯
  - 객체 구조 분해 할당(destructuring) 활용
-->
<mouse-position v-slot="{ x, y }">
  마우스 포지션: {{ x }}, {{ y }}
</mouse-position>
