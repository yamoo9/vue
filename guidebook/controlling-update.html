# 업데이트 관리

반응성 시스템에 의해 Vue 애플리케이션은 상태(data)가 변경되면 자동으로 업데이트 됩니다. 하지만 참조되는 데이터 타입인 배열, 객체의 경우 자동으로 업데이트 안되는 경우가 있으니 주의해야 합니다.

# 배열 관리

# 변경

배열 데이터의 상태가 변경됨을 감지하여 업데이트 하기 위해서는 배열 메서드를 사용해야 합니다.



 


 



methods: {
  addBook(book) {
    this.books.push(book);
  },
  removeBook(index) {
    this.books.splice(index, 1);
  }
}

# 대체

배열 데이터가 새로운 데이터로 교체된 경우, 계산된 속성을 사용해 변경된 데이터를 템플릿에서 사용합니다.



 



computed: {
  filteredDisplayItems() {
    return this.items.filter(item => item.display == '보임');
  }
}

주의!

배열 메서드를 사용하지 않고 아래와 같이 배열 아이템을 직접 수정하거나, 배열 길이를 바꿀 경우 반응성 시스템이 정상적으로 작동되지 않으니 주의해야 합니다.



 


 



methods: {
  addBook(book) {
    this.books[this.books.length] = book;
  },
  removeAllBooks() {
    this.books.length = 0;
  }
}

# 객체 관리

초기 data 속성에 할당된 객체의 속성이 아닌 경우, 반응성 시스템이 관리하지 않습니다.




 
 
 




 
 
 


data() {
  // 반응성 시스템이 관리합니다.
  return {
    reativity: {
      isReactive: true
    }
  }
},
methods: {
  // 반응성 시스템이 관리하지 않습니다.
  addProp(prop, value) {
    this.reativity[prop] = value;
  }
}

methods: {
  addReactivityProp(prop, value) {
    this.$set(this.reativity, prop, value);
  }
}

# 강제 업데이트

Vue 애플리케이션에서 강제 업데이트가 필요한 경우는 거의 없습니다. 하지만 강제 업데이트 필요한 경우, $forceUpdate 메서드를 사용하여 강제로 업데이트 할 수 있습니다.






 



methods: {
  addForceProp(prop, value) {
    this.reativity[prop] = value;
    // 강제로 업데이트 합니다.
    // 하지만 추가된 속성이 반응성 시스템에 관리되지는 않습니다.
    this.$forceUpdate();
  }
}
