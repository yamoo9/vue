# 계산된(Computed) 속성

# 계산된 속성을 사용하는 이유

템플릿 내에 식을 넣으면 편리합니다. 하지만 간단한 연산일 때만 이용하는 것이 좋습니다. 너무 많은 연산을 템플릿 안에서 하면 코드가 비대해지고 유지보수가 어렵습니다.



 
 
 
 
 
 
 



<time>
  {{
    new Date().toLocaleDateString().split('.').map((c,i) => {
      switch(i) {
        case 0: return `${c}년`;
        case 1: return `${c}월`;
        case 2: return `${c}일`;
      }
    }).join('');
  }}
</time>

템플릿 내에 복잡한 식을 사용하는 대신 계산된(computed) 속성을 사용하면 템플릿 내 코드를 간결하게 만들어 코드를 읽기 쉽게 만들어 줍니다.








 
 
 
 
 
 
 
 
 




<template>
  <time>{{ convertDayKR }}</time>
</template>

<script>
export default {
  computed: {
    convertDayKR() {
      return new Date().toLocaleDateString().split('.').map((c,i) => {
        switch(i) {
          case 0: return `${c}년`;
          case 1: return `${c}월`;
          case 2: return `${c}일`;
        }
      }).join('');
    }
  }
}
</script>

# 계산된 속성의 특징

계산된 속성의 장점은 연결된 상태(data) 값이 변경되면 자동으로 계산되어 화면을 업데이트 합니다.




 




 










export default {
  data() {
    return {
      date: new Date('2019-11-21')
    }
  },
  computed: {
    convertDayKR() {
      return this.date.toLocaleDateString().split('.').map((c,i) => {
        switch(i) {
          case 0: return `${c}년`;
          case 1: return `${c}월`;
          case 2: return `${c}일`;
        }
      }).join('');
    }
  }
}

convertDayKR: vm => vm.date.toLocaleDateString().split('.').map({
  // ...
})

# 메서드와의 차이점

메서드는 렌더링을 다시 할 때 마다 함수를 항상 실행하는 반면, 계산된 속성은 참조한 데이터 값이 변경될 때만 다시 계산되고 캐시(Cache) 되어 렌더링을 다시 해도 재 계산 없이 기억된 값을 사용합니다. 데이터 값이 변경될 때만 재 계산을 하고 싶은 경우 계산된 속성을 사용하는 것이 좋습니다.

간단한 예시 코드로 계산된 속성과 메서드를 비교해봅니다.

const vm = new Vue({
  el: '#app',
  template: `
    <div class="comapre-computed-methods">
      <p>계산된 속성: {{ compareC }}</p>
      <p>메서드: {{ compareM() }}</p>
    </div>
  `,
  data: {
    compareValue: 10
  },
  computed: {
    compareC() {
      // 계산된 속성은 참조하는 데이터 값이 변경되지 않으면 캐싱된 값을 사용합니다.
      console.log(`계산된 속성(캐싱): ${Date.now()}`);
      return this.compareValue;
    }
  },
  methods: {
    compareM() {
      // 메서드는 참조하는 데이터 값이 변경되지 않아도 재 실행되어 값을 반환합니다.
      console.log(`메서드: ${Date.now()}`);
      return this.compareValue;
    }
  }
});

# 계산된 Setter

계산된 속성은 메서드가 아닙니다. 이처럼 인자를 전달 실행하면 오류가 발생합니다.


 


<template>
  <time>{{ convertDayKR('2019-11-12') }}</time>
</template>
1
2
3

오류 발생!

Unexpected side effect in "convertDayKR" computed property.

계산된 속성에 값을 설정 하려면 set 함수를 사용합니다. v-on 디렉티브에 연결된 버튼을 클릭하면 계산된 속성이 참조하는 데이터(data) 값을 변경합니다. 값이 변경되면 재 계산된 결과를 UI에 업데이트 합니다.



 




<template>
  <div id="app">
    <button @click="convertDayKR = '2019-11-12'">
    <time>{{ convertDayKR }}</time>
  </div>
</template>

convertDayKR: {
  get() {

        case 0: return `${c}년`;
        case 1: return `${c}월`;

  },
  set(newDate) {
    this.date = new Date(newDate);

}
