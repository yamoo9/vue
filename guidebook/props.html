# props

# 속성 전달

부모 컴포넌트가 자식 컴포넌트로 child 데이터를 전달해야 한다고 합시다. 부모 컴포넌트는 자식 컴포넌트에 v-bind 디렉티브를 사용해 속성을 전달할 수 있습니다. 예제에서는 me 속성에 child 데이터를 값으로 전달합니다.

부모 컴포넌트






 



















<template>
  <div class="parent-component">
    <child-component
      v-for="(child,i) of children"
      :key="i"
      :me="child"
    />
  </div>
</template>

<script>
import ChildComponent from '@/components/Child';

export default {
  components: { ChildComponent },
  data: () => ({
    index: 0,
    children: [
      { name: 'props', description: '부모 ⇒ 자식 속성 전달' },
      { name: '$emit', description: '부모 ⇐ 자식 이벤트 방출' }
    ]
  })
}
</script>

자식 컴포넌트는 부모로부터 속성을 전달 받기 위해 컴포넌트 로직에 props 속성을 추가해야 합니다. props 속성은 배열 또는 객체를 설정합니다. 예제처럼 배열을 설정할 경우, 전달 받을 속성 이름을 문자로 설정합니다.

자식 컴포넌트

















 



<template>
  <div class="child-component">
    <template v-if="me">
      <h2>{{ me.name }}</h2>
      <p>{{ me.description }}</p>
    </template>
    <template v-else>
      <h2>자식 컴포넌트</h2>
      <p>부모 컴포넌트로 데이터 전달을 기다립니다...</p>
    </template>
  </div>
</template>

<script>
export default {
  // 부모 컴포넌트로 부터 속성을 전달 받음
  props: ['me']
}
</script>

# 전달된 속성 검증

부모로부터 전달 받을 속성을 구체적으로 기술하고자 한다면 객체를 사용합니다. 예를 들어 아래와 같이 props를 작성하면 전달 받을 속성 me는 String 데이터 타입만 허용하고, 다른 데이터 타입이 전달되면 오류를 출력합니다.



 



export default {
  props: {
    me: Object
  }
}

하나 이상의 데이터 타입을 허용하려면 배열을 사용해 허용할 데이터 타입을 설정할 수 있습니다. 아래와 같이 작성하면 Object 또는 Array 데이터 타입만 허용합니다.



 



export default {
  props: {
    me: [Object, Array]
  }
}

NOTE

JavaScript 데이터 타입 외에도 사용자가 정의한 클래스(또는 생성자 함수)를 통해 전달된 속성을 검증할 수 있습니다.

// 사용자 정의 클래스
class MyType { ... }

export default {
  props: {
    // 전달 속성이 사용자 정의 클래스의 인스턴스인지 검증
    myType: MyType
  }
}

# 필수 설정

필수적으로 부모 컴포넌트가 속성을 전달받아야 한다면? 아래와 같이 코드를 작성합니다.





 




export default {
  props: {
    me: {
      type: Object,
      required: true
    }
  }
}

# 기본 설정

필수 설정 대신, 기본 값을 설정하려면 아래와 같이 객체를 사용해 기술합니다.





 
 
 
 
 
 




export default {
  props: {
    me: {
      type: Object,
      default: function() {
        return {
          name: 'Child',
          description: 'Child 컴포넌트 입니다.'
        }
      }
    }
  }
}

# 사용자 정의 검증

필요한 경우 부모 컴포넌트로부터 전달된 속성이 유효한지 검증하는 검사(validator)를 사용자가 설정할 수 있습니다.




 
 
 
 
 



props: {
  me: {
    type: Object,
    validator(o) {
      return Object.keys(o).filter(prop => {
        return prop.match(/^name$/) || prop.match(/^description$/);
      }).length > 0;
    }
  }
}

# inheritAttrs 속성

일반적으로 컴포넌트 사용 시, 설정된 속성(사용자 정의 속성 포함)은 모두 상속(inheritance) 됩니다.

<navigation-link
  to="/profile"
  external="yes"
  content="프로필"
/>
1
2
3
4
5

navigation-link 컴포넌트 코드는 다음과 같이 부모가 전달한 속성을 받아 활용합니다.













 



<template>
  <a
    :href="to"
    :target="external === 'yes' ? '_blank' : null"
  >
    {{content}}
  </a>
</template>

<script>
export default {
  name: 'NavigationLink',
  props: ['to', 'external', 'content']
}
</script>

temp 변수는 HTML 표준 속성이 아닌데 실제 DOM에 반영되는 것은 문제가 됩니다. 이런 문제를 해결하려면 사용하지도 않을 temp 속성을 props에 설정해야 합니다.

props: ['to', 'external', 'content', 'temp']
1

하지만 이런 방법은 컴포넌트를 사용할 때 어떤 속성을 전달할지 모르므로 해결책이 될 수 없습니다. 이런 경우 inheritAttrs 속성 값을 false로 설정하면 컴포넌트는 props로 전달이 약속된 속성 외의 다른 속성을 상속 받지 않게 됩니다.

inheritAttrs: false,
props: ['to', 'external', 'content']
1
2

브라우저에서 결과를 확인해보면 전달된 속성만 잘 반영된 것을 확인할 수 있습니다.

<a href="/profile" target="_blank">프로필</a>
1

# $attrs 속성

앞서 다룬 inheritAttrs 속성 값을 false로 설정하여 속성을 상속 받지 않더라도 사용자가 설정한 임의의 속성은 컴포넌트의 $attrs 속성에 기억됩니다. beforeCreate 훅 함수를 사용해 이를 확인해 볼 수 있습니다.

beforeCreate() {
  console.log(this.$attrs);
  // { id: "nav-first", temp: "임시 변수" }
},
1
2
3
4

주의!

class, style 속성은 inheritAttrs 속성을 통해 상속을 해제해도 상속되며, $attrs 속성에 포함되지 않습니다.

이를 활용하면 props를 통해 전달이 약속된 속성이 아니라도 컴포넌트에서 사용할 수 있습니다.

<a :href="to" :id="$attrs.id">{{ content }}</a>
1

결과는 다음과 같이 id 속성이 반영된 결과를 보여줍니다.

<a id="nav-first" href="/profile" target="_blank">프로필</a>
1

하지만 속성이 여러 개라면 이러한 방법은 번거롭고 불편할 수 있습니다. 이런 경우 v-bind="$attrs"를 사용해 여러 속성을 일괄적으로 설정할 수 있습니다. (마치 속성이 상속된 것처럼)


 






<a
  v-bind="$attrs"
  :href="to"
  :target="external === 'yes' ? '_blank' : null"
>
  {{content}}
</a>

NOTE

<a
  v-bind="$attrs"
  v-on="$listeners"
>
  {{content}}
</a>
